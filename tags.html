Going into this class I knew nothing about coding and was frankly intimidated by the foreign nature of it, but it turns out that coding is not nearly as hard as I thought!

HTML is relatively straight forward and most of the work that I had to do when learning it was just the formatting; even the specific tags we use don't need to be completely

memorized since there are many resources online that provide the list of tags for you.


My first line of code was probably like most people's, which was simply printing out "Hello World!" It took me a little while to grasp the coding format, especially dealing with VSCode

and creating a repository, but eventually everything started to click. FreeCodeCamp was an excellent resource that I utilized as well to get better with HTML, since each exercise showed

you in real time what each tag was accomplishing on your webpage. I recommend FreeCodeCamp to anyone who is just starting to learn HTML like I was. Like the name implies,

it's free and all you need is an email. If you want to visit the site you can click here.




Now after learning the basics of HTML I felt a bit limited in my creativity when it came to text and overall webpage stylization. CSS came as a breath of fresh air

for me and made me even more excited to code, since it modifies HTML to allow custom fonts, colors, and webpage properties. Now I was customizing things that made my work look

like a real website. I could create borders and boxes that stuck to one part of the screen when scrolling, create custom webpage color schemes, and edit properties of text when

the mouse hovers over it. The process for implementing CSS is actually a lot easier than cramming class edits above and below your work on your index page. You link a style

page to your work and all of your customization is done on a separate page, which is much more organized and looks way cleaner in my opinion.
